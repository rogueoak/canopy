# 0022 — Card

## Problem

With the Twigs composition recipe established in [0020](0020-form-field.md), add the **Card**
Twig — the surface container molecule that frames related content (a stat, a form, a list, a
preview) on a raised surface. It is the layout primitive other compositions sit inside, and the
first Twig to exercise the **surface / elevation** tokens at the molecule layer.

Independently shippable: one composition, one PR. Follows the 0020 recipe — a compound
component on semantic tokens, adds no token.

## Outcome

- `@rogueoak/canopy/twigs` exports a themed `Card` family.
- A bordered, rounded, padded surface container with structured slots: `Card`, `CardHeader`,
  `CardTitle`, `CardDescription`, `CardContent`, `CardFooter`.
- Stories (basic content card · with header + footer actions · composing Seeds e.g. a form
  card with FormField/Button · both themes) and tests (renders slots, semantic heading,
  composes children).
- Storybook **Twigs** section updated.

## Scope

### In

- **Card** family — a presentational compound on semantic surface tokens:
  - `Card` — the container: `surface-raised` (or `surface`) fill, `border-border`, `rounded-lg`,
    the primitive `shadow-sm`, and a vertical structure. Forwards `ref`, spreads native `<div>`
    props.
  - `CardHeader` — top region; padded, stacks `CardTitle` + `CardDescription` with a small gap.
  - `CardTitle` — the card heading, rendered with a semantic typography role (`text-h3`/`h4`);
    renders a real heading element (default `<h3>`, overridable via `asChild`/`as` so the
    consumer keeps a correct document outline).
  - `CardDescription` — muted supporting text (`text-text-muted`, `text-body-sm`).
  - `CardContent` — the body region; padded, no top padding when it follows a header (the
    header owns the top inset).
  - `CardFooter` — bottom region for actions; padded, `flex items-center` with a gap.
  - Consistent padding scale across header/content/footer (e.g. `p-6`), so slots align.
- Stories: a basic content card; a card with header + footer buttons; a card composing Seeds
  (FormField + Button) to show molecules nesting — light + dark.
- Tests: renders each slot, `CardTitle` is a heading element with the right level, children
  compose, `className` overrides merge.

### Out

- **Interactive / clickable whole-card** (card-as-link, hover elevation, selection state) —
  a later variant; this is the static container.
- **Elevation token** — there is no semantic elevation token yet (Select/Tooltip use the
  primitive `shadow-*`); Card does the same. Introducing a semantic elevation scale is its own
  follow-up, not this spec.
- **Media/image slots** (`CardMedia`, aspect-ratio cover images) — follow-up if needed.

## Approach

Follows the **0020 recipe**: a compound component composing into a structured surface, styled
with semantic tokens only (`bg-surface-raised`, `border-border`, `text-text`,
`text-text-muted`, the typography roles), `cn()` for merging, no new token, theme-agnostic by
construction — the raised surface themes through the token layer exactly as the portalled Seeds
do. Each slot is a small `forwardRef` `<div>` (or heading) spreading native props. Vitest +
Testing Library verify structure and the heading semantics, not pixel padding.

## Acceptance

- [ ] `Card`, `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`, `CardFooter`
      exported from `@rogueoak/canopy/twigs`; semantic tokens only, light **and** dark, no
      per-component theme code.
- [ ] Card is a bordered, rounded, shadowed raised surface with consistent slot padding.
- [ ] `CardTitle` renders a real heading element (default `<h3>`) with a semantic type role and
      is overridable for outline correctness.
- [ ] `CardDescription` is muted supporting text; slots compose arbitrary children incl. Seeds.
- [ ] Each part forwards `ref` and spreads native props; `className` merges via `cn()`.
- [ ] Stories cover basic / header+footer / composing-Seeds in both themes; tests
      (slots + heading semantics + composition + className merge) pass.
