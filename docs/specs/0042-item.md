# 0042 - Item

## Problem

Canopy frames blocks of content with `Card` (0022) and lays out row-shaped navigation with
`Breadcrumb` (twig), but it has no generic **row/list-item primitive**: the horizontal
`[media] [title + description] [actions]` shape that every list, menu, settings row, notification
entry, or search result reuses. Today a consumer building any of those has to hand-roll the flex
layout, the icon/avatar column, the two-line title-plus-description text stack, the trailing
actions cluster, and the hover/link affordance from scratch every time - re-deriving spacing,
alignment, and truncation on each surface. shadcn/ui only recently added an `Item` primitive for
exactly this reason, and canopy has the same gap: `Card` is a padded vertical surface, not a
compact horizontal row, so it is the wrong tool for a dense list.

This is for anyone assembling a **list of things**: file lists, member/assignee rows, menu and
command entries, notification feeds, settings togg'-rows, and search results. `Item` owns that row
shape once - as a presentational Twig that composes with the atoms canopy already ships
(`Avatar`, `Badge`, `Button`, `Spinner` 0017) - so those surfaces stop re-implementing it.

## Outcome

- A new canopy component family, `Item`, exported from `@rogueoak/canopy/twigs`, that renders a
  horizontal row with three regions: leading **media**, a flexible **content** column (title +
  description), and trailing **actions**.
- **Parts**: `Item` (the row container), `ItemMedia` (icon/avatar column), `ItemContent` (the
  flexible middle column), `ItemTitle`, `ItemDescription`, and `ItemActions` (trailing cluster) -
  each a small `forwardRef` element that spreads native props and merges `className` via `cn()`.
- **Variants**: `variant` of `default` (transparent, for use inside another surface), `outline`
  (bordered standalone row), and `muted` (a `bg-muted` filled row). Layout is consistent across
  variants; only the surface treatment changes.
- **`asChild`** on `Item` renders the row as a single caller-provided element (Radix `Slot`), so
  the whole row becomes a link (`<a>`) or a `<button>` while keeping the row layout and gaining a
  hover affordance - the common "clickable list row" case - without `Item` owning any interaction
  state itself.
- **Layout behaviour**: `ItemMedia` sits at the row's leading edge and does not shrink;
  `ItemContent` flexes to fill and its `ItemDescription` truncates/wraps predictably;
  `ItemActions` pins to the trailing edge. Vertical alignment is centered by default.
- **A11y**: purely presentational by default (a plain row of `<div>`s carries no role); semantics
  come from what the caller composes in (`asChild` link/button, an enclosing `role="list"`/`<ul>`,
  `ItemTitle` as a real heading via its own `asChild`). Guarded by observable tests, not scaffolding.
- **Theming**: styled with the 0005 recipe (full-literal semantic-token Tailwind utilities, `cn()`
  merge, `forwardRef` + native prop spread), so it themes light/dark through the token layer with
  no `dark:` on the common path.
- **Docs**: a Storybook catalog entry (playground, variants, as-link, with-avatar, with-actions, a
  list of items) and canopy README + `overview/` living docs updated on completion.

  on completion.

### Out
- **Interactive state / selection** (selected/active/checked list rows, roving-tabindex list
  keyboard nav) - `Item` is presentational; a selectable `ListItem`/`Menu` Branch that owns
  interaction state is a clean follow-up.
- **A `List` container** component (the `role="list"` wrapper with dividers/spacing) - deferred;
  `Item` composes into any list the caller provides for v1.
- **Size variants / density scale** - single comfortable density for v1; a `size` prop can be added
  later without breaking the surface.
- **Swipe / drag-reorder affordances** - out of scope; a later interactive spec.
- Changing `Card` (0022), `Breadcrumb`, or any other existing component - `Item` is purely
  additive; no existing public API changes.

## Approach

**Primitive stack: plain elements + Radix `Slot`, styled with the 0005/0020 Twigs recipe.** `Item`
is a presentational compound with no interaction state and no portal, so it is a **Twig** (composes
a layout of parts via sibling elements, mirroring `Card`), not a Branch. It needs no Radix
behaviour primitive - only `@radix-ui/react-slot` (already a canopy dependency) to power `asChild`.
No new runtime dependency is introduced, so there is no new dependency surface for
security/architecture review here.

**Part surface (mirrors the shadcn Item, canopy-styled).**
- `Item` - the row container: `flex items-center gap-3` with padding, `rounded-lg`, and a
  cva-driven `variant` surface (`default` transparent; `outline` adds `border border-border`;
  `muted` adds `bg-muted`). When `asChild` is set it renders through `Slot` onto the caller's
  element (an `<a>` or `<button>`), which also gets a `hover:bg-muted-raised` affordance so a
  clickable row highlights; the focus-visible ring tokens apply so a link/button row is keyboard
  visible. `Item` forwards `ref` and spreads native props.
- `ItemMedia` - the leading icon/avatar column: `flex shrink-0 items-center justify-center`, sized
  to hold an `Avatar` or a Lucide icon; does not shrink when the row is tight.
- `ItemContent` - the flexible middle column: `flex min-w-0 flex-1 flex-col` (the `min-w-0` lets
  its text truncate inside the flex row) with a small vertical gap.
- `ItemTitle` - the primary line: `text-label text-text` (the `cn()` typography-role rule keeps both
  classes). Renders a `<div>` by default; `asChild` promotes it to a real heading when the caller's
  outline needs one.
- `ItemDescription` - the secondary line: `text-body-sm text-text-muted` with `truncate` so a long
  description clips to one line by default (caller can override via `className`).
- `ItemActions` - the trailing cluster: `ml-auto flex shrink-0 items-center gap-2`, pinned to the
  row's trailing edge, holding `Button`/`Badge`/icon-button actions.

**Styling & recipe.** FULL LITERAL token utility strings so Tailwind v4's scanner emits each
utility, `cn()` merge with caller `className` winning, `forwardRef` + native prop spread on every
part, semantic tokens only, and no `dark:` on the common path - light/dark flips through the token
layer exactly as `Card` (0022) does. The clickable-row hover uses `bg-muted-raised` per the
raised-surface rule.

**Accessibility.** `Item` intentionally carries no ARIA role by default - it is a layout primitive,
and imposing a role would fight the many contexts it drops into. Semantics are composed by the
caller: `asChild` makes the row a real `<a>`/`<button>` (focusable, keyboard-activatable, with the
focus-visible ring), `ItemTitle asChild` promotes the title to a heading, and a surrounding
`<ul>`/`role="list"` gives the list its structure. Tests assert the observable outcomes (the
`asChild` element is an `<a>`/`<button>` and its `href`/`onClick` work; a heading appears when
`ItemTitle asChild` is used), per the repo learning that a11y is guarded by outcomes.

**Motion.** None - `Item` is a static layout primitive; the only interactive feedback is the
token-based hover background on a clickable row.
