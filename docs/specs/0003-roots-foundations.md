# 0003 — Roots: Foundations (palette, type, spacing)

## Problem

The skeleton (0002) proved the token pipeline with a throwaway sample token. Now we define
the **real foundation** — the colour, typography, and spacing that every Canopy component
will style against. This is the "get it right before building components" milestone: once
locked, the whole system inherits it, so the values must be deliberate and tuned visually,
not guessed.

Direction (decided with the developer): **muted & natural** palette — desaturated, earthy,
warm neutrals; a **moss/olive** signature brand green (~`#5A6638`); **modern-minimal**
typography — one clean humanist/grotesque sans. Light-theme values are defined here; the
dark-theme remap and runtime switching are **0004**.

Audience: us (building components on top) and future rogueoak app teams consuming the tokens.

## Outcome

When done:

- `@rogueoak/roots` ships a real, two-tier token system — **primitive** ramps + **semantic**
  tokens — generated by Style Dictionary into CSS vars, the typed TS export, and the Tailwind
  v4 preset (the 0002 pipeline, now with real content).
- Components can style entirely against **semantic** tokens (`color-bg`, `text-default`,
  `border`, `color-primary`, `radius-md`, `space-4`, `text-lg`, `shadow-md`, …) — never raw
  ramp values.
- The Storybook **Foundations** section renders the system as a living spec: full colour
  ramps + semantic swatches, a type specimen, the spacing scale, radii, elevation, and
  motion — so we tune and **lock** it visually.
- Light theme is complete and AA-contrast-checked for text roles. Dark theme structure is in
  place (semantic layer) but values are deferred to 0004.

## Scope

### In
- **Colour primitives** (ramps, 50–950): `moss` (brand), `bark` (warm brown), `stone` (warm
  neutral/gray), `amber` (accent), plus functional `success` / `warning` / `danger` / `info`
  tuned earthy. Perceptually-even steps; documented contrast for text-bearing steps.
- **Semantic colour tokens** (light theme mapping): surfaces (`bg`, `surface`,
  `surface-raised`, `overlay`), text (`text-default`, `text-muted`, `text-subtle`,
  `text-inverted`), lines (`border`, `border-strong`, `ring`), roles (`primary` +
  `-foreground`, `secondary`, `accent`, `muted`), and status (`success`/`warning`/`danger`/
  `info` + foregrounds).
- **Typography**: one sans family — **Figtree** (variable) — + a mono (**Geist Mono**) for
  code; a type scale (xs→6xl), weights, line-heights, tracking; semantic text roles (display,
  h1–h4, body, body-sm, label, caption, code). **Self-hosted** via `@fontsource` (open-licensed)
  — no external CDN.
- **Spacing**: 4px base modular scale (`space-0…space-32`).
- **Radii**: `none, sm, md, lg, xl, 2xl, full`.
- **Elevation**: `shadow-sm/md/lg/xl` — soft, slightly warm.
- **Motion**: durations (`fast/base/slow`) + easings (standard/emphasized/decelerate).
- **Storybook Foundations** stories rendering all of the above for visual review.
- Update README (Tokens section reflects the real system) and tick `0003`.

### Out
- **Dark theme** values + runtime theme switching → **0004**.
- Any components (Seeds) → **0005**.
- Layout/breakpoint tokens, density modes, multi-brand theming → later.
- Real `npm publish` → later.

## Approach

- **Two tiers.** Primitive ramps are the raw palette; semantic tokens reference them
  (`color-primary → {color.moss.600}`), so theming is a remap of the semantic layer — exactly
  the reference seam hardened in 0002. Components consume **only** semantic tokens.
- **Authoring.** DTCG token JSON in `packages/roots/tokens/` split by concern
  (`color/primitive`, `color/semantic`, `typography`, `space`, `radius`, `shadow`, `motion`).
  Ramps tuned for even perceptual steps and AA contrast where text sits.
- **Type roles** compose family + size + weight + line-height + tracking into semantic styles
  so components reference `text-h2`, not raw values.
- **Visual lock.** The Storybook Foundations stories ARE the review surface — we iterate hexes,
  type, and spacing there until they're right, then lock. The spec's acceptance is satisfied
  by your sign-off in Storybook, not by the numbers in this doc.

### Approved starting palette (developer-reviewed in `.preview/foundations.html`)

The eight ramps from the preview are the agreed starting point — carried into the DTCG
sources, then contrast-tuned in Storybook:

- `moss` (brand): `#F4F6EA #E7EBCF #D0D8A6 #B6C17E #9DA95C #828F46 #5A6638 #48522D #394123 #2D331D #171B0E`
- `bark` (warm brown): `#F8F4EF #EFE5D9 #DECBB2 #C9AC8A #B38B63 #976E48 #7C5A3B #634730 #4E3927 #3E2E20 #241A12`
- `stone` (warm neutral): `#F7F5F1 #EDEAE3 #DAD5CA #C2BBAC #A39B8A #857C6B #6B6354 #554E42 #423D34 #322E28 #1E1B16`
- `amber` (accent): `#FBF4E9 #F5E6CB #EACB98 #DDAC66 #CF9343 #C2873B #A06C2E #7D5326 #5E3F20 #46301B #28190D`
- `success` / `warning` / `danger` / `info` — desaturated functional ramps as previewed.

Type: **Figtree** (sans) + **Geist Mono**; scale 12→60px on a ~1.2 ratio; spacing 4px base;
`radius-md` 0.5rem.

### Decisions (locked)
- **Sans family: Figtree** (variable, open-licensed). Mono: Geist Mono.
- **Self-host** fonts via `@fontsource` — no CDN.
- **Functional colours desaturated** to sit within the earthy palette (no default-bright alerts).
- Palette mood **muted & natural**, brand green **moss/olive**.
